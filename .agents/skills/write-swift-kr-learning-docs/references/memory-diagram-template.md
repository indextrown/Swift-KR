# Swift-KR 메모리 다이어그램 템플릿

Swift의 실행 메모리와 ARC 소유 관계를 설명할 때 사용하는 SVG 규칙이다. 강의 자료의 화면을 그대로 옮기지 말고, 학습할 관계만 남긴 개념 모델로 작성한다.

## 사용 순서

1. `assets/memory-diagram-template.json`을 문서의 에셋 폴더 아래 `specs/`로 복사한다.
2. 제목, 설명, 열, 노드, 연결선을 주제에 맞게 수정한다.
3. 다음 명령으로 SVG를 생성한다.

```bash
node .agents/skills/write-swift-kr-learning-docs/scripts/render-memory-diagram.mjs \
  docs/{문서-경로}/assets/specs/{이름}.json \
  docs/{문서-경로}/assets/{이름}.svg
```

4. SVG의 `<title>`과 `<desc>`가 그림 없이도 관계를 설명하는지 확인한다.
5. SVG를 `.memory-diagram` 컨테이너로 감싸 모바일에서는 그림 내부만 가로로 스크롤되게 한다.
6. Rspress 페이지를 데스크톱과 모바일 폭에서 열어 글자 겹침과 문서 전체의 가로 넘침을 확인한다.

```mdx
<div className="memory-diagram">

![그림만 보지 않아도 관계를 이해할 수 있는 대체 텍스트](./assets/{이름}.svg)

</div>
```

## 레이아웃 규칙

- `columns` 배열 순서가 화면의 열 순서다. `code`, `data`, `heap`, `stack` 타입을 사용한다.
- 각 열의 `width`는 독립적으로 지정한다. 객체가 많으면 힙을 넓히고, 호출 프레임이 길면 스택을 넓힌다.
- 스택이 여러 개면 `stack` 타입 열을 반복하고 제목을 `스택 1`, `스택 2`처럼 구분한다.
- 코드·데이터·힙은 노드 묶음을 기본적으로 세로 가운데에 정렬한다. 비교 순서가 중요하면 `nodeLayout.vertical`을 `top`으로 바꾼다.
- 스택은 아래에서 위로 쌓는다. `nodes` 배열의 첫 번째 노드가 가장 아래에 놓이고 다음 노드가 그 위에 쌓인다.
- 노드의 `x`는 자신이 속한 열의 본문 왼쪽을 기준으로 한다. 자동 배치에서는 `y`를 작성하지 않는다.
- 노드의 `id`는 전체 그림에서 고유해야 한다. 연결선의 `from.id`, `to.id`가 이 값을 사용한다.
- 생성기는 카드 제목, 행, 푸터가 안전 여백을 넘으면 정해진 최소 크기까지 글자를 자동으로 줄인다.
- 최소 크기로도 들어가지 않으면 생성을 중단한다. 이때 `width`를 늘리거나 문구를 줄이고, 텍스트 위치를 테두리 가까이 밀어 해결하지 않는다.

`nodeLayout`은 열 안의 노드 묶음을 배치한다.

| `vertical`  | 배치 방식                                               |
| ----------- | ------------------------------------------------------- |
| `center`    | 노드 묶음을 세로 가운데에 놓는다. 코드·데이터·힙 기본값 |
| `top`       | `paddingTop`부터 아래로 쌓는다.                         |
| `bottom-up` | `paddingBottom`부터 위로 쌓는다. 스택 기본값            |
| `manual`    | 각 노드의 `y`를 직접 사용한다. 겹침 표현에만 제한한다.  |

```json
{
  "id": "stack",
  "title": "스택",
  "type": "stack",
  "width": 320,
  "nodeLayout": {
    "vertical": "bottom-up",
    "gap": 24,
    "paddingBottom": 20
  },
  "nodes": [
    {
      "id": "main-frame",
      "kind": "frame",
      "x": 16,
      "width": 288,
      "height": 132
    },
    {
      "id": "callee-frame",
      "kind": "frame",
      "x": 16,
      "width": 288,
      "height": 148
    }
  ]
}
```

## 노드 타입

| `kind`     | 용도                                     |
| ---------- | ---------------------------------------- |
| `object`   | 힙에 생성된 클래스 인스턴스              |
| `closure`  | 힙에 유지되는 저장 클로저와 캡처 리스트  |
| `frame`    | 함수 호출 프레임과 지역 변수             |
| `code`     | 실행 코드나 함수 본문                    |
| `data`     | 전역·정적 데이터 또는 예제에서 비운 영역 |
| `note`     | 해제 순서, 판단 기준 같은 보조 설명      |
| `released` | 해제되었거나 더 이상 연결되지 않은 상태  |

`rows`는 `{ "label": "프로퍼티", "value": "값" }` 형태다. 강조가 필요하면 `tone`을 `strong`, `weak`, `danger`, `safe`, `muted` 중 하나로 지정한다.

텍스트의 기본 크기와 자동 축소 하한은 다음과 같다.

| 요소       | 기본 | 최소 |
| ---------- | ---: | ---: |
| 카드 제목  | 16px | 12px |
| 행 이름·값 | 14px | 11px |
| 카드 푸터  | 13px | 11px |

제목과 태그가 같은 줄에 있으면 태그 너비와 둘 사이의 간격을 먼저 확보한다. 행은 왼쪽 이름과 오른쪽 값 사이에 최소 16px 간격을 남긴다.

노드의 왼쪽 컬러 라인은 별도 장식 막대가 아니라 카드 테두리의 일부로 렌더링한다. 같은 위치에 기본 테두리와 컬러 막대를 겹쳐 그리지 않는다. 컬러 테두리가 왼쪽의 둥근 모서리를 담당하고 위·오른쪽·아래 기본 테두리와 자연스럽게 이어지게 한다.

## 연결선 규칙

| `kind`     | 표현                  | 의미                                      |
| ---------- | --------------------- | ----------------------------------------- |
| `strong`   | 파란 실선 화살표      | 대상을 소유하고 수명을 유지하는 강한 참조 |
| `weak`     | 보라 점선 화살표      | 대상을 소유하지 않는 약한 참조            |
| `layout`   | 회색 점선 화살표      | 코드·타입·실행 위치 사이의 구조 연결      |
| `released` | 회색 취소선 또는 점선 | 제거되었거나 끊어진 참조                  |

연결점은 `left`, `right`, `top`, `bottom` 중 하나를 고른다. `offset`으로 해당 변의 시작점부터 떨어진 거리를 지정할 수 있다. 두 연결선이 겹치면 `curve`의 네 값을 조정한다.

```json
{
  "from": { "id": "owner", "side": "bottom", "offset": 80 },
  "to": { "id": "child", "side": "top", "offset": 80 },
  "kind": "strong",
  "label": "강한 참조",
  "curve": [0, 70, 0, -70],
  "labelOffset": [52, 0]
}
```

## 교육 자료에서 지킬 점

- 주소와 참조 수는 실제 디버거 결과가 아니라 개념을 설명하기 위한 값이라고 본문에서 밝힌다.
- 스택과 힙의 배치는 이해를 위한 단순화이며 최적화에 따라 실제 저장 위치가 달라질 수 있음을 설명한다.
- 빈 코드·데이터 영역을 억지 예제로 채우지 않는다. 해당 예제에서 사용하지 않으면 `예제에서는 생략`으로 표시한다.
- 화살표 방향은 “참조를 가진 쪽 → 참조되는 대상”으로 통일한다.
- 색만으로 의미를 구분하지 않는다. 실선·점선과 연결선 라벨을 함께 사용한다.
